gctree.dag_filtering.filter_dag(dag, priority_weights=None, verbose=False, outbase=None, mutability_file=None, substitution_file=None, isotypemap=None, isotypemap_file=None, idmap=None, idmap_file=None, isotype_names=None, img_type='svg')[source]ÔÉÅ

Filter trees in the provided history DAG.

Trim the provided history DAG, containing sequence labels, by-node abundance attributes, and a parameters attribute, to minimize a linear combination of branching process likelihood, isotype parsimony score, mutability parsimony score, and number of alleles, with coefficients provided in the argument priority_weights, in that order.

To compute each tree‚Äôs combined score, a linear transformation is applied to each trait, so that the best observed value is mapped to 0, and the worst observed value is mapped to 1. The tree score is the weighted sum of transformed traits, using passed coefficients.

Parameters

  • dag (HistoryDag) ‚Äì A history DAG object to trim

  • priority_weights (Optional[Sequence[float]]) ‚Äì A sequence of coefficients for prioritizing tree weights. The order of coefficients is: branching process likelihood, isotype parsimony score, mutability parsimony score, and number of alleles. If priority_weights is not provided, trees will be ranked lexicographically by traits, in the same order.

  • verbose (bool) ‚Äì print information about trimming

  • outbase (Optional[str]) ‚Äì file name stem for a file with information for each tree in the DAG, and rank plots of likelihoods. If not provided, no such files will be written.

  • mutability_file (Optional[str]) ‚Äì A mutability model

  • substitution_file (Optional[str]) ‚Äì A substitution model

  • isotypemap (Optional[Mapping[str, str]]) ‚Äì A mapping of sequences to isotypes

  • isotypemap ‚Äì A dictionary mapping original IDs to observed isotype names

  • isotypemap_file (Optional[str]) ‚Äì A csv file providing an isotypemap

  • idmap (Optional[Mapping[str, Set[str]]]) ‚Äì A dictionary mapping unique sequence IDs to sets of original IDs of observed sequences

  • idmap_file (Optional[str]) ‚Äì A csv file providing an idmap

  • isotype_names (Optional[Sequence[str]]) ‚Äì A sequence of isotype names containing values in isotypemap, in the correct switching order

  • img_type (str) ‚Äì Format to be used for output plots, if outbase string is provided.

Returns

The trimmed history DAG, containing all optimal trees according to specified criteria.

A slower but more numerically stable equivalent to ll_genotype_dagfuncs.

Parameters

  • q (float64) ‚Äì branching process parameters

Returns

A historydag.utils.AddFuncDict object, which can be unpacked and used as keyword arguments for DAG trimming and weight annotation according to branching process likelihood. Weight format is (log likelihood, cmcounts) where cmcounts is a FrozenMultiset containing abundance, mutant clade count pairs. To use these functions for DAG trimming, use an optimal function like lambda l: max(l, key=lambda ll: ll[0]) for clarity, although min or max should work too.

gctree.dag_filtering.mutability_dagfuncs(*args, **kwargs)[source]ÔÉÅ

Return functions for counting mutability parsimony on the history DAG.

Returns a historydag.AddFuncDict containing functions for counting tree-wise sums of mutability distances in the history DAG.

This may not be stable numerically, but we expect every tree to have a unique mutability parsimony score for non-degenerate mutability models, so so this shouldn‚Äôt matter in practice.

Arguments are passed to MutationModel() constructor.

gctree.dag_filtering.mutability_distance(mutation_model)[source]ÔÉÅ

Returns a fast distance function based on passed mutation_model.

First, caches computed mutabilities for k-mers with k // 2 N‚Äôs on either end. This is pretty fast for k=5, but the distance function should be created once and reused.

The returned distance function sums \(-log(mutability * p)\) over all sites which do match between its two sequence arguments, where mutability is the mutation frequency of the k-mer surrounding the mutated base (in the first sequence argument) and p is the transition probability to the new base.

Note that, in particular, this function is not symmetric on its arguments.

Return functions for filtering by isotype parsimony score on the history DAG.

Returns a historydag.utils.AddFuncDict which contains functions necessary for filtering by isotype parsimony score. These functions return weights which are a tuple containing a progressive isotype score, and a frozenset containing isotypes used internally to compute isotype scores.

  • idmap_file (Optional[str]) ‚Äì A csv file providing an idmap

  • isotype_names (Optional[Sequence[str]]) ‚Äì A sequence of isotype names containing values in isotypemap, in the correct switching order

Returns

A historydag.utils.AddFuncDict which may be passed as keyword arguments to historydag.HistoryDag methods which trim or annotate by weight. These functions return weights which are a tuple containing a progressive isotype score, and a frozenset containing isotypes used for computing isotype scores.
